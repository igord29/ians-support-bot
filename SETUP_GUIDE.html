1
Create a Telegram Bot
5 min ▾
  1. Open Telegram and search for @BotFather
  2. Send: /newbot
  3. Follow prompts — give it a name like "Ian's Agent" and a username like ian_field_agent_bot
  4. Copy the HTTP API token — save as TELEGRAM_BOT_TOKEN
  5. Search for @userinfobot and message it to get your personal user ID — save as TELEGRAM_CHAT_ID and ALLOWED_TELEGRAM_USER_IDS
💡 The allowed user IDs list prevents anyone else from using your bot if they somehow find it.
2
Set Up Google Service Account
15 min ▾

This gives the bot permission to write to your Calendar, Tasks, and Gmail.

  1. Go to console.cloud.google.com → New Project → name it "Ian Agent"
  2. Enable these APIs: Google Calendar API, Tasks API, Gmail API
  3. Go to IAM & Admin → Service Accounts → Create Service Account
  4. Name it "ian-agent", click Create. Skip optional fields.
  5. Click the service account → Keys tab → Add Key → JSON → Download
  6. Copy client_email → GOOGLE_SERVICE_ACCOUNT_EMAIL
  7. Copy private_key → GOOGLE_SERVICE_ACCOUNT_KEY
⚠️ For Gmail sending: Service accounts need "Domain-wide delegation" in Google Workspace Admin. If you're on personal Gmail, use OAuth2 instead — ask Claude to swap out gmail.js with nodemailer + OAuth2.

Share your calendar with the service account email (give it "Make changes" permission) in Google Calendar settings.

3
Deploy to Railway
10 min ▾
  1. Push the code to a GitHub repo
  2. Go to railway.app → New Project → Deploy from GitHub
  3. Select your repo
  4. Go to Variables tab and add all keys from .env.example
  5. Railway auto-deploys and gives you a public URL like https://your-app.up.railway.app
npm run setup-webhook # Pass your Railway URL as argument if needed: # WEBHOOK_URL=https://your-app.up.railway.app node scripts/setup-webhook.js
💡 Railway free tier = $5/month credit which covers this easily. Alternatively use Render.com free tier (has cold starts — less ideal for a bot).
4
Set Your Reminder Schedule
2 min ▾

The bot is pre-configured with this schedule (all times EST):

☀️ 8:00 AM — Morning briefing (calendar + high priority tasks) ⏰ 10:00 AM — Pending items check-in (if any open tasks) ⏰ 12:00 PM — Midday check-in ⏰ 2:00 PM — Afternoon nudge ⏰ 4:00 PM — Pre-evening check 📋 6:00 PM — Evening digest (full list + ideas captured)

Edit the cron schedules in src/index.js to adjust timing.
